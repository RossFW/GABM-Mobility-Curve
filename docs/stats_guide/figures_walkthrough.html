Part 1 — The Two Models

Every diagnostic in Figures 24a and 24b is a check on one of two regression models you fit to each LLM config's 20,000 binary rows. This section gives you just enough about each model to make the diagnostics make sense.

1.1 Fixed-Effects Logit (the simpler one)

The formula you run in R:

glm(stay_home ~ infection_pct + I(infection_pct^2) + factor(agent_id),
    family = binomial, data = df)

factor(agent_id) expands into 99 dummy variables, one for each agent except a reference agent. Each of those dummies gets its own freely-estimated coefficient. The model has 3 "shared" parameters (intercept, infection linear, infection squared) plus 99 agent-specific offsets = 102 total parameters.

The linear predictor (what you run through the sigmoid) for agent i at infection level x:

ηi(x) = β0 + αi + βinf·x + βinf²·x²

where αi is agent i's personal dummy (zero for the reference agent).

Worked example — Opus 4.5, agent 0 (reference agent), at 3.2% infection

Say the fitted coefficients are β₀ = -2.1, βinf = 0.95, βinf² = -0.08, α0 = 0 (reference). Then:

  • η = -2.1 + 0 + 0.95×3.2 + (-0.08)×3.2² = -2.1 + 3.04 - 0.82 = +0.12
  • P(stay home) = 1 / (1 + e-0.12) ≈ 0.530

For a different agent (say agent 42 with α42 = +0.8), the same infection level gives η = -2.1 + 0.8 + 3.04 - 0.82 = 0.92 → P ≈ 0.715. Agent 42 is "stay-home prone" and their dummy captures that.

What we can and cannot estimate in this model

Can estimate: the infection curve shape (βinf, βinf²) and each agent's personal baseline (αi).

Cannot estimate: any effect of between-agent features — traits, age, gender. Why? Because each agent has a fixed trait profile. Putting extraverted in the model alongside factor(agent_id) would create perfect multicollinearity: extraversion is perfectly determined by agent ID. R would drop one or the other.

This is why we report only the infection coefficients from Model 1. The 99 agent dummies are statistical bookkeeping you don't interpret.

1.2 Random-Effects Logit (what is a random intercept, really?)

The formula:

glmer(stay_home ~ infection_pct + I(infection_pct^2) +
                  male + extraverted + agreeable + conscientious +
                  emot_stable + open_to_exp + age_years +
                  (1 | agent_id),
      family = binomial, data = df)

The key piece is (1 | agent_id). This is saying: "every agent still gets their own intercept offset, but those 100 offsets aren't freely estimated — they're assumed to come from a single Normal distribution."

ui ~ Normal(0, σ²agent)   for i = 1, 2, ..., 100

So where the fixed-effects model had 99 completely independent αi coefficients, the random-effects model treats the ui values as 100 draws from a shared bell curve. The model has to simultaneously estimate:

  1. The shape of the bell curve — specifically, σagent, the spread of the agent-offset distribution.
  2. Each agent's position in that bell curve — u1, u2, ..., u100.
  3. The fixed-effect coefficients — intercept, infection terms, traits, age, gender.
Key idea The crucial difference from FE: FE fits agent dummies independently (each agent's αi is whatever best fits that one agent's data). RE fits them jointly under a shared distributional constraint. That constraint lets us estimate trait effects.

Why this unlocks trait estimation

In the FE model, putting extraverted alongside agent dummies was impossible because extraversion is constant within agent. In the RE model, the agent-specific ui aren't freely estimated — they're shrunk toward zero via the Normal constraint. This leaves statistical "room" for trait coefficients. Now βextraverted can capture the average effect of extraversion across the pool of agents, while ui captures whatever is left over after accounting for that agent's observable traits.

Worked example — Shrinkage / partial pooling, in pictures

Suppose agent 42 has 200 observations with 160 "stay home" (so 80%). The agent's observed log-odds = log(160/40) = +1.39.

In the FE model, α42 will be fit so that the model's predicted mean equals 80% — independent of other agents. The model has no way to "doubt" this estimate because agent 42 is their own island.

In the RE model, the estimate for u42 is "shrunk" toward 0. How much depends on σagent:

  • If σagent = 0.5 (agents don't vary much in baseline), u42 might end up at, say, +0.4 (heavy shrinkage).
  • If σagent = 2.0 (agents vary a lot), u42 might end up at +1.2 (less shrinkage).

This is called partial pooling: the model "borrows strength" across agents. Outliers get pulled back, but only partially — the data still dominates when it's strong.

For Opus 4.5, the R output gives σ̂agent = 1.90, meaning agents vary substantially in baseline stay-home tendency even after controlling for traits.

The linear predictor for the RE model

For agent i at infection level x, the linear predictor is:

ηi(x) = β0 + βinf·x + βinf²·x² + βmale·malei + βext·extravertedi + ... + βage·agei + ui

And as before, P(stay_home) = 1 / (1 + e−η).

Worked example — same agent, RE model

For Opus 4.5, suppose the coefficients are:

  • β0 = −1.50, βinf = 0.42, βinf² = −0.05
  • βmale = 0.20, βextraverted = −0.30, βagreeable = +0.15, βconscientious = +0.25, βemot_stable = +0.10, βopen = −0.05, βage = 0.01
  • Agent 42: female, extraverted, agreeable, conscientious, emotionally stable, not open, age 34. BLUP u42 = +0.18.

At infection 3.2%:

η = −1.50           (intercept)
  + 0.42×3.2 + (−0.05)×3.2²   (infection curve = +0.83)
  + 0.20×0                     (female, no male contribution)
  + (−0.30)×1                  (extraverted → −0.30)
  + 0.15×1 + 0.25×1 + 0.10×1   (agreeable, consc., emot. stable = +0.50)
  + (−0.05)×0                  (not open)
  + 0.01×34                    (age = +0.34)
  + 0.18                       (random intercept)
  = −1.50 + 0.83 − 0.30 + 0.50 + 0.34 + 0.18 = +0.05

P(stay home) = 1 / (1 + e^(−0.05)) ≈ 0.51

Agent 42 has ~51% stay-home probability at 3.2% infection under this model.

What it actually means to say “ui ~ Normal(0, σ²)”

This is a claim about the shape of a histogram you could draw if you could somehow observe the 100 latent ui values. Nothing more, nothing less.

Each agent has some ui — the leftover stay-home tendency specific to that agent, after the model accounts for their age, gender, and Big Five traits. You can't observe ui directly (only 0/1 decisions are observed), but conceptually each agent has one. If you collected all 100 of them into a list and plotted a histogram, the Normal assumption says it should look bell-shaped, centered at 0, with spread σ.

The bars are a hypothetical histogram of 100 ui values. The red curve is the Normal(0, σ²) density the model assumes they follow. For Opus 4.5, σ̂_agent = 1.90, meaning about 68% of agents have ui between −1.9 and +1.9 on the log-odds scale.

Generative vs. inferential Your intuition — “we're sampling starting points from Normal(0, σ²) and extracting trait impact given those random starts” — is right as a generative description of how the model imagines your data was produced. If nature were rolling dice:
(1) draw 100 ui values from Normal(0, σ²);
(2) compute each agent's log-odds at each infection level using fixed-effect coefficients + ui;
(3) flip a Bernoulli coin at that probability for each row.

In practice you're running this backward. You have the 20,000 coin flips. You estimate σ², the trait coefficients, and implied positions of each ui all simultaneously from the data. You never actually sample anything during inference — the sampling language is just the generative story. This distinction matters because it's tempting to picture an "order" (first the ui, then the traits) but both are estimated jointly.

Where σ² comes from — how it's estimated

Your instinct might be: "just compute the variance of the 100 ui values — that's σ²." Problem: you don't observe the ui values. Only 0/1 outcomes. Each ui is latent (unobserved). So σ² has to be estimated some other way.

glmer estimates it via maximum likelihood: it searches for the value of σ² (and all fixed-effect coefficients) that makes your observed binary data most probable under the model. The thing being maximized looks like this:

L(β, σ²) = ∏(100 agents) ∫ (likelihood of this agent's 200 rows | ui) × Normal(ui | 0, σ²) dui

In English: for each agent, integrate over every possible value of their ui, weighted by (a) how plausible that ui is under the Normal(0, σ²) assumption and (b) how well that ui would explain their 200 observed rows. Multiply that across all 100 agents. The (β, σ²) pair that maximizes this product is your estimate.

Why the integral is hard (and what “Laplace” does)

For linear mixed models (Gaussian outcomes) the integral has a closed form — easy. For logit GLMMs (Bernoulli outcomes), it doesn't. lme4 uses a Laplace approximation: for each agent, it finds the peak of the integrand, then approximates the full integrand by a Gaussian centered at that peak — and the Gaussian can be integrated analytically. This turns the intractable integral into an approximate one that's fast to compute. The optimizer (bobyqa) then hill-climbs over (β, σ²) using these approximate log-likelihoods until it finds the joint maximum.

Why var(BLUPs) ≠ σ̂² exactly

After glmer finishes, it reports σ̂ and the 100 BLUPs (estimated ui values). It's tempting to compute var(BLUPs) and expect it to equal σ̂². It won't — two reasons:

  1. Circularity: BLUPs depend on σ² (they're shrunken toward 0 by a factor that itself involves σ²). You can't derive σ² from BLUPs because you need σ² first to compute them.
  2. Shrinkage biases var(BLUPs) downward: BLUPs are shrunken estimates, so their sample variance systematically underestimates the true σ². Maximum likelihood corrects for this by integrating over the uncertainty in each ui rather than treating the point estimate as the truth.

For Opus 4.5, σ̂_agent = 1.90 while sd(BLUPs) ≈ 1.80. Close but not equal, with BLUPs slightly shrunken. The gap grows if each agent has fewer observations (because less data per agent = more shrinkage needed).

How we verify the Normal assumption actually holds

After estimation, you end up with σ̂ plus the 100 BLUPs. The Normal assumption is then testable: does the histogram of BLUPs actually look Normal?

Two standard checks:

  1. Random-intercept Normal Q-Q plot (the 3rd view in Figure 24b). Sort the 100 BLUPs, standardize by σ̂, and plot each against the theoretical Normal quantile for its rank. Points near the diagonal → Normal shape fits. Systematic curvature → the shape is wrong (bimodal, skewed, heavy-tailed).
  2. Shapiro-Wilk test (optional). A formal hypothesis test of Normality. Like KS it's over-powered at large N — visual inspection of the Q-Q plot is the primary diagnostic.

What to do if the assumption visibly fails:

  • Mild deviation (slight curve at tails, small S-shape): usually harmless. Your β̂ estimates stay unbiased; uncertainty is slightly off.
  • Severe bimodality: might signal a missing categorical predictor that splits agents into groups. Add it as a fixed effect.
  • Heavy tails: switch to a t-distribution for the RE (requires a different package like glmmTMB).
  • Skew: indicates an asymmetric source of agent-level variation. Usually acceptable in practice.

For your 21 configs, inspect the Q-Q plots in Figure 24b. Most configs show mild S-shapes at the tails (normal for N=100 agents — the tail estimates are noisy). No config shows catastrophic non-Normality, so the assumption is fine in practice.

1.3 Assumptions of each model

Assumption Fixed-Effects Random-Effects
Outcome is Bernoulli given the linear predictor ✓ Required ✓ Required
Logit link correctly specified ✓ Required ✓ Required
Continuous predictors enter log-odds linearly ✓ Only infection ✓ Infection + age
Observations independent given the predictors ✓ Required ✓ Required (given RE)
Random intercepts follow Normal(0, σ²) — (n/a) ✓ Required
Random intercepts uncorrelated with predictors — (n/a) ✓ Required

The RE model has all the FE assumptions plus two new ones: Normality of the random intercepts, and their independence from observed predictors. This is why RE needs more diagnostic work — there's more structure to validate.

1.4 How each model is fit (briefly)

Fixed-effects logit (glm)

R's glm uses iteratively reweighted least squares (IRLS) to find coefficients that maximize the likelihood. Starting from an initial guess, it repeatedly refits a weighted linear regression until convergence. For N = 20,000 and 102 parameters this takes a few seconds.

Random-effects logit (glmer)

The RE model can't be fit with plain IRLS because the likelihood involves integrating over the distribution of ui. lme4::glmer uses a Laplace approximation (by default) — it approximates the integral via a second-order Taylor expansion around its mode. For each candidate value of the fixed-effect coefficients and σagent, the optimizer evaluates this approximate likelihood; a numerical optimizer (you use "bobyqa") then searches for the maximum. Typical runtime on your data: 10–150 seconds per config, depending on how hard the likelihood surface is to navigate.

Why is glmer sometimes so much slower than glm?

The fixed-effects model has a convex log-likelihood with a unique maximum. Plain IRLS finds it quickly. The random-effects likelihood is not always well-behaved — the variance components can be near zero or near infinity, the optimization surface can have flat regions, and the Laplace approximation itself is more expensive than IRLS. Convergence issues force extra iterations. This is why glmer benefits from warm starts and careful optimizer control (bobyqa, maxfun = 100000, etc.).

Part 2 — Figure 24a breakdown (Fixed-Effects Logit Validation)

Figure 24a is a 3×7 grid of small panels, one per LLM configuration. The toggle cycles between three views. Each view tests something different.

2.1 Calibration view — "pred vs obs"

What it checks: when the FE model says the average probability of staying home is p, does the actual observed rate come out around p?

How each panel is built

1Fit the FE logit for the config. Save the fitted model object.
2For every one of the 20,000 rows, ask R for the model's predicted probability: predict(m1, type = "response"). Each row now has a p̂i.
3Group the 20,000 rows by infection_level (40 bins, 500 rows per bin).
4Within each bin, compute:
  • mean predicted P = mean of the 500 p̂i values
  • observed rate = fraction of rows with stay_home = 1
5Plot the 40 points: x = mean predicted P, y = observed rate. Add diagonal line (perfect calibration).
6Summary statistic: MAD = mean of |predicted − observed| across the 40 bins, in percentage points.
Worked example — one bin (infection = 0%, Opus 4.5)
  • 500 rows at infection = 0% (100 agents × 5 reps).
  • Each row has some p̂i under the FE model. Since the FE model's prediction at infection=0 is β₀ + αi, this varies across agents.
  • Mean of those 500 p̂i = 0.045 (4.5%).
  • Observed rate at infection = 0%: 0 / 500 = 0.000 (0%).
  • Point plotted at (0.045, 0.000) — just below the diagonal.

Doing this for all 40 infection bins gives you one panel. A well-calibrated model has all 40 points close to the diagonal.

Why it works The FE model's per-agent dummies absorb each agent's personal baseline perfectly within-agent, so calibration at the aggregated-bin level is really testing whether the shared quadratic infection curve matches the average infection response. A visible sag or bulge in this panel means the quadratic is wrong — you'd want to try a cubic or a spline.

2.2 DHARMa residual Q-Q view

What it checks: do the model's residuals, after a proper randomization trick, look like draws from Uniform(0,1) — which they should if the model is correctly specified?

This is the conceptually trickiest plot, so let's build it slowly.

Step 1 — Why "residuals" are weird for binary data

In ordinary least squares, a residual is just observed − predicted and you can plot those against theoretical Normal quantiles to check the Normality assumption. For binary data, residuals can only take two values per row: 1 − p̂ (if observed = 1) or −p̂ (if observed = 0). Those aren't Normal-distributed — they can't be, they're discrete. Standard residual plots don't apply.

Step 2 — The Probability Integral Transform (PIT) idea

There's a classical result: if X is a continuous random variable with CDF F, then F(X) is Uniformly distributed on [0, 1]. In words: "plugging an observation into its own CDF gives a Uniform(0,1) result."

So if our binary outcomes were continuous and we knew the true model, we could compute F(observed) for each row, check that those values pile up uniformly on [0, 1], and call it a day. But our data is discrete. The CDF of Bernoulli(p̂) is a step function:

For an observed value of 0, F "jumps through" the interval [0, 1−p̂]. For an observed value of 1, it jumps through [1−p̂, 1]. There's no single number that represents F(observed); it's an entire interval.

Step 3 — The randomized PIT fix

DHARMa's fix: for each observation, sample a Uniform random value from its jump interval. Call that value the scaled residual for that row.

  • Observed = 0, predicted p̂ = 0.3 → residual ~ Uniform(0, 0.7)
  • Observed = 1, predicted p̂ = 0.3 → residual ~ Uniform(0.7, 1.0)

And here's the magic: under a correctly specified model, every single observation contributes a Uniform(0,1) draw to the pooled residual distribution.

The density-math proof (optional)

Take an observation with true probability p. It contributes:

  • with probability p, Uniform(1−p, 1) — density 1/p on that interval
  • with probability 1−p, Uniform(0, 1−p) — density 1/(1−p) on that interval

Multiply each case by its probability of occurring:

  • density contribution on [1−p, 1]: p × (1/p) = 1
  • density contribution on [0, 1−p]: (1−p) × (1/(1−p)) = 1

The density stitches to exactly 1 across all of [0, 1]. That's Uniform(0, 1). And this is true no matter what p is.

Step 4 — How DHARMa actually implements it via simulation

The math above assumed we knew the true p for each observation. We don't — we only know the fitted p̂. DHARMa handles this by:

1Simulate 250 synthetic datasets from the fitted model. For each original row, draw 250 Bernoulli outcomes using its p̂.
2For each row, count the proportion of simulated outcomes that are strictly less than the observed value, and the proportion less than or equal. These define the jump interval.
3Sample one Uniform random value from that jump interval. That's the scaled residual for the row.
4Repeat for all 20,000 rows. You now have 20,000 scaled residuals.
Worked example — one observation

Row: agent 42 at infection 3.2%, observed = 1. FE model predicts p̂ = 0.30.

  • DHARMa simulates 250 Bernoulli(0.30) draws. Roughly 75 are 1, 175 are 0.
  • P(simulated < 1) = proportion of simulated values strictly less than 1 = 175/250 = 0.700
  • P(simulated ≤ 1) = 1 (everything is ≤ 1).
  • Jump interval for observed = 1: [0.700, 1.000].
  • Draw Uniform(0.700, 1.000) — say this draw = 0.824. That's the row's scaled residual.

The residual lands in the "upper tail" because the row had an outcome the model thought was unlikely. That's fine — under a correct model, this happens about 30% of the time (= p̂), which is exactly enough to keep the pooled distribution uniform.

Step 5 — Building the Q-Q plot

1Sort the 20,000 scaled residuals from smallest to largest.
2Compute 99 summary quantiles: the 1st, 2nd, ..., 99th percentiles of the sorted residuals.
3Plot each quantile at its theoretical Uniform position: the 30th percentile should be around 0.30 if the model is correct.
4Add the diagonal y = x. Departures from this line = model misspecification.
Interpretation A tight hug of the diagonal = well-specified model. Any systematic curve indicates a distributional mismatch, which is a fingerprint of what's wrong. An S-curve usually means dispersion issues. A consistent sag or bulge means the model is systematically over- or under-predicting at some probability range.

2.3 Residuals vs infection view

What it checks: is there a systematic pattern in residuals across infection levels? (If so, the quadratic curve is missing something.)

1Start from the 20,000 scaled residuals computed in 2.2 (DHARMa output).
2Group them by infection_level (40 bins, 500 residuals per bin).
3For each bin, compute the mean residual and its standard error.
4Plot bin mean vs infection level. Add a horizontal reference line at 0.5 (the expected value under a correct model).

Under a correctly specified model, DHARMa scaled residuals are Uniform(0, 1), so their expected mean is 0.5. Any systematic deviation of the bin means from 0.5 — a tilt, a curve, a bowl — is evidence that the infection effect isn't captured right.

What a "bad" plot would look like

If the true infection response is cubic but you fit only a quadratic, your residuals at the middle infection levels would have a positive bias (model under-predicts), while residuals at the ends would have a negative bias. You'd see a gentle arch — high in the middle, low at the edges. That shape is a red flag that you need higher-order polynomial terms.

Part 3 — Figure 24b breakdown (Random-Effects Logit Validation)

Figure 24b has all the Figure 24a views plus two new ones (Random-Intercept Normal Q-Q, Residuals vs Age) because the RE model has additional assumptions to validate. The same-named views (Calibration, DHARMa Q-Q, Residuals vs Infection) work identically but with Model 2's predictions.

3.1 Calibration view (Model 2)

Mechanically identical to 2.1, but the predicted probability per row uses the full random-effects model: fixed-effect coefficients + the agent's BLUP ui. The in-browser computation uses the fixed effects averaged across the 100 agent profiles at each infection level, which approximates the per-row average within a bin. The result for well-calibrated configs looks near-identical to Model 1's — aggregation at the bin level averages out most of the between-model differences.

3.2 DHARMa Q-Q view (Model 2)

Identical recipe to 2.2, but the simulations incorporate the fitted σagent and the estimated fixed effects, so the null distribution DHARMa compares against is the richer RE model. Under a correct RE specification, residuals still pool to Uniform(0, 1).

One subtlety: DHARMa can simulate residuals in several modes for GLMMs. The default ("conditional") conditions on the estimated random intercepts — so it's checking whether residuals are uniform given those BLUPs. Any systematic misfit in this view points to problems in the fixed-effect part of the model (infection curve, trait effects, etc.), not the random-effects part.

3.3 Random-Intercept Normal Q-Q view (unique to Model 2)

What it checks: are the 100 estimated agent random intercepts actually Normally distributed, as the model assumes?

How the panel is built

1After fitting the RE model, extract the 100 BLUPs: u = ranef(m2)$agent_id. These are the point estimates of u1, ..., u100.
2Also extract the model's estimated σ̂agent (the RE standard deviation). For Opus 4.5 this is σ̂ = 1.90.
3Standardize: zi = ui / σ̂agent. Now the zi values should be roughly Standard Normal (mean 0, variance 1) if the assumption holds.
4Sort the z values from smallest to largest.
5Compute theoretical Standard Normal quantiles at positions (i − 0.5)/n for i = 1, ..., 100. Each agent's theoretical quantile is Φ⁻¹((i − 0.5)/100).
6Plot pairs (theoretical z, observed z). Add the diagonal. Straight line along diagonal = Normal assumption holds.

(Values illustrative; actual numbers depend on the config.) Plotting (theoretical, observed) for all 100 pairs — if they fall near the diagonal, the Normal assumption is fine. A curve that bends away from the diagonal at the extremes means heavy tails (observed more extreme values than Normal predicts) or light tails (less extreme than Normal).

What deviations mean

  • S-shape with tails above diagonal on both ends: observed BLUPs have heavier tails than Normal. Some agents have very extreme baselines the Normal assumption misses. This could bias σ̂ upward.
  • S-shape with tails below diagonal: observed BLUPs are lighter-tailed than Normal — less extreme than the model expects. This is harmless in practice.
  • One-sided skew (most points above or below diagonal): the RE distribution is skewed rather than symmetric. The mean-zero assumption is still respected (BLUPs sum to zero by construction), but the shape is asymmetric.
Practical note With 100 agents (your sample size), the Normal Q-Q plot has decent resolution for detecting gross misspecifications but can look noisy at the tails. Don't over-interpret a single outlier dot — look for systematic curvature. The Shapiro-Wilk test is another option, but like KS, it's over-powered at this N; visual inspection is primary.

3.4 Residuals vs Age / Residuals vs Infection

What they check: do residuals vary systematically with age or infection? (If so, the linear/quadratic form for that predictor is inadequate.)

1Take the 20,000 DHARMa scaled residuals from the RE model (3.2).
2For the Age view: bin by 5-year age groups (18–22, 23–27, ..., 63–67 → 10 bins). For the Infection view: bin by infection level (40 bins).
3Compute mean and standard error of residuals within each bin.
4Plot bin mean vs predictor. Reference line at 0.5.

The interpretation is the same as 2.3 — a flat line at 0.5 means the predictor's functional form is adequate. For Age: a linear βage is assumed, so a tilt would suggest non-linearity. For Infection: a quadratic is assumed, so a systematic pattern would suggest higher-order curvature.

Practical example — what "max |Δ|" tells you

The footer of each panel shows max |Δ| — the largest deviation of any bin's mean from 0.5. If this number is:

  • < 0.03: well within noise. Residuals are flat.
  • 0.03–0.05: mild departure. Usually fine, but worth glancing at the panel shape.
  • > 0.05 (shown in red): one or more bins deviate meaningfully. If these are at the edge of the x-range (small n) it's often noise; if they're consistent across a range, that's real misspecification.

For your current output, most configs land under 0.05. Red flags concentrate in specific LLMs with underdispersion (Sonnet 4.0, GPT-5.2 off) and reflect residual-variance violations rather than missing curvature per se.
